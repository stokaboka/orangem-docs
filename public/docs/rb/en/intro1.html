Description

Rb for information systems with web interface and SQL or NoSQL data sources. The basis of Rb is the application server Impress Application Server and SPA web application that implements the user interface. In a nutshell, developing a web application for working with databases using Rb is reduced to the following steps:

  1. Obtaining a database schema: tables and their structure, links (foreign keys);
  2. Including in the empty model the selected tables (data sources);
  3. Applying links to selected data sources from the database schema (optional);
  4. add missing links and / or delete unnecessary links;
  5. setting properties of data sources and field / column properties: headers, filters and sorts by default, the ability to edit and save data, etc;
  6. Adjustment of fields / columns of data sources for visual components for displaying and / or editing data;
  7. Creating forms and applications;
  8. Assign a model name and save.

As a result of these steps, a web application will be created allowing you to view and edit data in accordance with the settings. When the selected record is modified, the links are processed. This will download the records to the linked data sources.

At the moment, the main UI component of RbTable is used to display data source rows. RbTable allows you to view, sort and filter records. In the testing process, the RbTreeTable component

Editing of records is made in the form generated for each data source. You can use the components RbList, RbComboBox, RbListOfValues, RbSequencer, RbDataMap, RbLinearProgress, RbColorIcon to change field / column values (select values) and / or filter records.

Basic elements of Rb

The application server performs the following functions:
  • is a bridge between the web application and various data sources;
  • process requests for application to data sources;
  • performs service operations;
  • is a web server for distributing static files to the web application.
The web application implements the user interface for working with data sources. The structure of the web application consists of several parts:
  • model;
  • application;
  • the form;
  • data source;
  • links between data sources;
  • data bindings.
The model stores a description of the information domain of the information system and:
  • determines the sources of data, their structure and relationships;
  • defines logically completed applications;
  • defines forms (user interface);
  • determines the relationship between forms and applications;
  • determines the relationship of data sources.
The application combines forms into a logically completed part of the information system:
  • description of the application;
  • the order of the application;
  • collection of forms;
  • order of display and availability of forms.
The form is the main element of the user interface. Contains visual components associated with the data sources:
  • form description;
  • form layout;
  • collection of data sources;
  • distribution of visual components in the form grid.
The data source describes the structure of a single database object (usually a table):
  • general information about the database and the table;
  • possibility of reading / writing;
  • sorting and filters by default;
  • record validators;
  • hierarchical structure;
  • columns of the table (data source fields).
The column of the table (the data source field) determines the type stored in the source (table) of the data:
  • description;
  • possibility of reading / writing;
  • sorting and filters by default;
  • visual components for displaying and editing data.
The links between the data sources make it possible to update the data and visual representation in the linked data sources when the record in the parent data source is changed. Data binding allows data sources to be used to populate fields in other data sources (lists of values) and / or filtering when displaying data. Transactions: execution of all actions with data (upload, change, links) are performed using transactions.

The scheme

RB Core

The core of Rb is the kernel: Rb::Core, which manages the data and their representations based on the models developed by the user. Rb::Core performs the following functions:
  • loading the selected model;
  • the formation of the object model by controlling the loading and changing of data, the work of links and data binding;
  • formation of the object model of visual representation.
